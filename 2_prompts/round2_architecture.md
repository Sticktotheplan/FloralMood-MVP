# Round 2 - Architecture

Invite utilisateur pour définir l'architecture technique et le stack technologique.



« Passe en mode Architecte Technique. Propose une architecture logicielle globale pour l’application : quels seront les principaux composants du système (frontend, backend, base de données, API externes, workflows automation…) et comment vont-ils interagir ? »



« Précise la stack technologique pour chaque composant (langages, frameworks, bibliothèques) et justifie ces choix pour un développement rapide du MVP (ex: facilité, rapidité, compatibilité avec l’IA ou n8n). »



« Décris comment les décisions d’architecture assurent la scalabilité et maintenabilité minimales du projet : quelles bonnes pratiques adoptons-nous dès le MVP pour éviter de tout refondre plus tard (ex: modularité, séparation des responsabilités, sécurité de base) ? »



« Indique enfin les intégrations externes à prévoir au niveau de l’architecture : par exemple, utilisation d’un service d’hébergement d’images, intégration d’une API d’IA, ou utilisation de n8n en complément du backend, et la façon dont cela s’intègre dans l’architecture. »
